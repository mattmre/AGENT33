## Key Architectural Decisions

- Reuse AGENT-33's current hook framework for Claude Code lifecycle integration rather than introducing a separate hook model.
- Extend the current approval service into tokenized stateless approvals for MCP instead of building a second HITL subsystem.
- Add dynamic tool activation and semantic skill resolution on top of the existing registries, with session-aware state.
- Import EVOKORE skills through the pack/skill ecosystem so they remain tenant-aware, governable, and marketplace-compatible.
- Treat the current voice daemon as a scaffold to be replaced or wrapped by a true sidecar implementation in the final phase.
