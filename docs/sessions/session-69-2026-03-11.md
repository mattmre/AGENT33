## Findings

- A stale wrap PR can become useful again if it is repurposed to the current handoff instead of being closed blindly; preserve durable guidance, but rewrite queue snapshots aggressively.
- Phase 45 approval tokens needed a real verify-then-consume path in governed execution; consuming the token before consuming the approval record created a state-consistency bug.
- Track 2’s catalog service needed tenant context only for plugin-backed data; the builtin tool registry and skill registry remain intentionally system-scoped.
- `asyncio.get_event_loop().run_until_complete(...)` is no longer safe in tests that run on Linux/Python 3.11 without a default loop; `asyncio.run(...)` is the correct test helper for these sync REST tests.
- Test-only module overrides need to win over stale `app.state` services. In the tool catalog routes, preferring `_catalog_service` over `app.state.tool_catalog_service` fixed a full-suite isolation failure that small targeted runs did not expose.
- Class-level `PropertyMock` helpers on `MagicMock` are too leaky for broad pytest runs; plain stub objects are safer for catalog metadata tests.
