## Executive Summary

Camunda 8 is a mature, enterprise-grade process orchestration platform built around the Zeebe workflow engine. Unlike AGENT-33's documentation-only approach, Camunda provides a complete runtime implementation with BPMN 2.0 and DMN standards. Their architecture emphasizes horizontal scalability, fault tolerance via append-only logs, and event-driven messaging patternsâ€”concepts highly applicable to AGENT-33's orchestration specifications.

The repository demonstrates sophisticated operational patterns including backpressure management, four-pillar observability (logs, metrics, traces, profiles), CQRS-style read/write separation, and comprehensive testing pyramids. These patterns can inform AGENT-33's specification evolution without requiring implementation changes, aligning with the model-agnostic and evidence-first principles.

Key opportunities for AGENT-33 include adopting Camunda's event scope model for trigger management, implementing their backpressure/rate-limiting specifications, and adapting their BPMN-based visual workflow representation for documentation. Camunda's exporter architecture and connector patterns offer proven specifications for extensibility that could enhance AGENT-33's integration guidance.

## Repository Overview

### Purpose
Camunda 8 orchestrates complex business processes spanning people, systems, and devices. The repository contains:
- **Zeebe**: Cloud-native process engine (core workflow execution)
- **Tasklist**: Human task orchestration
- **Operate**: Real-time process monitoring and troubleshooting
- **Optimize**: Process analytics and constraint identification
- **Identity**: Authentication and authorization management

### Technology Stack
| Layer | Technology |
|-------|------------|
| Language | Java 21+ (primary), gRPC/REST APIs |
| Persistence | Append-only log (RocksDB), RDBMS support (MyBatis/Liquibase) |
| Messaging | Netty-based TCP messaging, Kafka integration |
| Serialization | MessagePack, Protocol Buffers, SBE |
| Observability | Prometheus metrics, OpenTelemetry traces, structured logging |
| Containerization | Docker, Kubernetes-native |

### Key Concepts

#### Event-Driven Architecture
Camunda processes events as first-class citizens:
- **Event Scopes**: Define visibility and resolution semantics
- **Triggers**: Mechanisms for activating catch events (publication, propagation, cancellation)
- **Event Scope Instances**: Runtime state for tracking trigger eligibility

#### BPMN 2.0 Standards
- ISO-standard visual process modeling
- Decision Model and Notation (DMN) for business rules
- FEEL (Friendly Enough Expression Language) for expressions

#### Distributed Consensus
- Atomix-based cluster membership and messaging
- Raft consensus protocol for partition replication
- Append-only journal for durability

## Feature Inventory

### Feature 1: Backpressure Management
**Description**: Configurable rate limiting with multiple algorithms (Vegas, AIMD, Gradient, Gradient2, Fixed) to prevent broker overload. Commands are dropped rather than buffered to maintain system stability.

**Application to AGENT-33**:
- Specify backpressure patterns for agent orchestration
- Define whitelisting rules (critical commands bypass limits)
- Document semaphore/concurrency limits in `parallel/SEMAPHORE_CONTROL.md`

**Implementation Pattern**:
```yaml
# orchestrator-backpressure.schema.json
backpressure:
  enabled: true
  algorithm: "vegas"  # vegas | aimd | fixed | gradient | gradient2
  useWindowed: true
  whitelist:
    - intent: "CANCEL"
    - intent: "COMPLETE"
    - intent: "FAIL"
```

**Priority**: High
**Effort**: Medium (2-3 days specification work)
